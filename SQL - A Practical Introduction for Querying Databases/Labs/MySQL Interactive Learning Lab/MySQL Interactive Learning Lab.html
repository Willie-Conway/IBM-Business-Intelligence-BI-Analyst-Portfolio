CREATE TABLE

Creating Database Tables

Basic Syntax:
CREATE TABLE table_name ( column1 datatype constraints, column2 datatype constraints, column3 datatype constraints, ... );
Example: Create a COUNTRY table
CREATE TABLE COUNTRY ( ID INT NOT NULL, CCODE CHAR(2), NAME VARCHAR(60) );
DROP TABLE Syntax:
-- WARNING: This permanently deletes the table and all its data DROP TABLE table_name; -- Example: DROP TABLE COUNTRY;
⚠️ Important: Always ensure you don't need the table data before using DROP. For development, it's common to use: DROP TABLE IF EXISTS table_name;

ALTER TABLE

Modifying Table Structure

Add a Column:
-- Syntax: ALTER TABLE table_name ADD COLUMN column_name datatype constraints; -- Example: Add phone number to author table ALTER TABLE author ADD COLUMN telephone_number BIGINT;
Modify Column Data Type:
-- Change BIGINT to CHAR(20) to store phone numbers with formatting ALTER TABLE author CHANGE telephone_number telephone_number CHAR(20);
💡 Tip: Use CHAR/VARCHAR for phone numbers to store formatting characters like +, (, ), and -.
Drop a Column:
-- Remove a column from a table ALTER TABLE author DROP COLUMN telephone_number;
TRUNCATE TABLE (Delete All Rows):
-- Syntax: TRUNCATE TABLE table_name; -- Example: Delete all data from author table TRUNCATE TABLE author;
⚠️ Difference from DELETE: TRUNCATE is faster but can't be rolled back. DELETE is slower but can use WHERE clause.

SELECT Queries

Retrieving Data from Tables

Basic SELECT Syntax:
-- Select specific columns SELECT column1, column2, column3 FROM table_name; -- Select all columns SELECT * FROM table_name;
Using WHERE Clause:
-- Filter rows based on conditions SELECT * FROM COUNTRY WHERE ID < 5; -- Filter with string comparison SELECT * FROM COUNTRY WHERE CCODE = 'CA'; -- Multiple conditions SELECT * FROM CUSTOMERS WHERE country = 'USA' AND age > 21;

INNER JOIN (Most Common):
-- Get orders with customer information SELECT o.order_id, o.order_date, o.total_amount, c.first_name, c.last_name, c.email FROM ORDERS o INNER JOIN CUSTOMERS c ON o.customer_id = c.customer_id;
LEFT JOIN (All from Left Table):
-- Get all customers and their orders (if any) SELECT c.customer_id, c.first_name, c.last_name, o.order_id, o.order_date FROM CUSTOMERS c LEFT JOIN ORDERS o ON c.customer_id = o.customer_id;
RIGHT JOIN (All from Right Table):
-- Get all orders and customer info (if customer exists) SELECT o.order_id, o.order_date, c.first_name, c.last_name FROM ORDERS o RIGHT JOIN CUSTOMERS c ON o.customer_id = c.customer_id;

Practice: Chicago Data Analysis

Using the Chicago database tables, write a JOIN query to:

Find schools in communities with high per capita income:

SELECT s.Name_of_School, e.Per_Capita_Income
FROM chicago_public_schools s
INNER JOIN chicago_socioeconomic_data e
ON s.Community_Area_Name = e.Community_Area_Name
WHERE e.Per_Capita_Income > 30000;
SELECT * FROM schools, income
WHERE schools.area = income.area
AND income > 30000;
JOIN schools TO income
GET school_name, income
WHERE income > 30000;
